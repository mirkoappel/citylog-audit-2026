# memory/

Persistente Wissensbasis des Forschungsagenten. Wird zu Beginn jedes neuen
Kontextfensters gelesen um nahtlos weitermachen zu können.

| Datei | Inhalt |
|---|---|
| `MEMORY.md` | Index aller Memory-Dateien |
| `project.md` | Projektziel, harte Regeln, aktueller Stand, nächste Schritte |
| `feedback.md` | Erwartungen des Nutzers, Arbeitsweise, was zu vermeiden ist |

## Wann aktualisieren

Nach jeder Forschungsphase: `project.md` mit aktuellem Stand und nächsten Schritten
aktualisieren. `feedback.md` aktualisieren wenn der Nutzer neue Hinweise zur
Zusammenarbeit gibt.

## Was hier nicht steht

Experiment-Ergebnisse, Techniken-Details und Forschungsbefunde stehen in `research/`.
Memory enthält nur was über Kontextfenster hinaus stabil bleibt: Projektziel,
Regeln, grober Stand, Arbeitsweise.
